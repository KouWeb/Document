# テンプレートコーディングルール

## 基本的なコーディングルール

-   インデントはタブ（サイズ 4）を使用してください。半角スペースは使用しないでください。
-   サイト内のリソースへのアクセス（CSS や JS ファイルの読み込み、リンク、画像など）はルートパス（/から始まる表記）を使用してください。
-   UTF-8 でエンコードされていて、文字化けなどが起きない場合は、記号を文字参照に変換する必要はありません。
-   英数字は基本的には半角に統一してください。
-   記号やスペースは文脈によって半角・全角を使い分けてください。

## HTML 関連

-   可能な限りセマンティックタグを使用してください。
-   文書構造的に意味のないマークアップをしないでください（`<div>`の中に直接テキストなど）。
-   スタイルを付与する場合は必ず class 属性を使用してください。
-   アンカーは id 属性を使用してください。name 属性は使用しないでください（廃止済）。
-   文脈上意味のある画像には、必ず alt 属性を記述してください。
-   文脈上意味がなく装飾的な役割の画像には、必ず空の alt 属性を記述してください（alt 属性を省略しないでください）。
-   デバイスサイズによって異なる画像を読み込ませる場合は、`<picture>`及び`<source>`要素を使用してください。
-   `<img>`要素、`<source>`要素に width や height を記述してください。

### なぜ alt を省略せず空にしておくのか？

読み上げソフトの動作に違いがあります。

alt 属性がある場合  
alt 属性の内容が読み上げられる

alt 属性がない場合  
画像ファイル名が読み上げられる

alt 属性が空の場合  
読み飛ばされる

alt 属性を省略すると画像ファイル名が読み上げられてしまい、かえって冗長になり意味が分かりづらくなるため、意味を持たない画像でも alt 属性は省略しないでください。

## CSS/SASS 関連

### 基本

-   エンコードの指定（`@charset "UTF-8";`）は記述する必要はありません（CSS はデフォルトで UTF-8）。
-   SCSS 記法を使用してください。
-   特別な理由がない限り、マジックナンバー（固定値）の使用は避けてください。特に要素の幅や高さなど、レイアウトに影響する部分での使用には注意してください。使用する必要がある場合は、何の値なのかわかるようにコメントを残してください。
-   モジュール毎に、なんのモジュールなのかを CSS コメント`/* */`で記載してください。
-   無効化したいコードやメモとして残しておきたいコードなど、コンパイル後は不要になるコードには SASS コメント`//`を使用してください。また、不要になったコードは削除してください。
-   CSS コメントと SCSS コメントを使い分けてください。コンパイル後も残しておきたいコメント（モジュール名など）は CSS コメントを使用してください。コンパイル後に不要になるコメントアウト（テスト用のコードなど）は SCSS コメントを使用してください。

### セレクター関連

-   FLOCSS のプレフィックスに従った命名をしてください。
-   MindBEMding に従った命名をしてください。
-   単語の接続はケバブケースを使用してください。
-   スタイルの定義に id セレクターは使用しないでください。
-   スタイルを付与する要素には必ず class を指定してください。
-   タイプセレクターを使用しないでください。
-   原則、子孫セレクターなど他の要素に依存するセレクターを使用しないでください。
-   状態の変化を制御するモディファイア用 class には is-プレフィックスを付与してください。
-   モディファイア名を除き、色や形など見た目を表現する命名はしないでください。
-   JS フックにスタイルを付与しないでください。

### 設計

-   リセット CSS を SASS に含めないでください。別途ファイルを読み込ませてください。

## JavaScript 関連

-   変数・定数名は基本的にはキャメルケースを使用してください。
-   定数で固定値を指定する場合は、定数名を大文字にしてください。この場合、単語の接続はスネークケースにしてください。
    -   `const MAX_WIDTH = 300;`
